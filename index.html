Bootstrap Force Device

Force Bootstrap's grid to display as if on another device. Load .force-bootstrap.css into your document, and layout your columns as you normally would. Then add .force-* to an ancestor of the grid classes, and the CSS will do the rest.

Forces the grid to behave as if it's on the device of your choosing .force-xs, .force-sm, .force-md or .force-lg

.col-lg-8
.col-lg-4
.col-md-6
.col-sm-2
.col-xs-6
.col-xs-4
.col-xs-2

Just add .force-xs (or sm, md, or lg) to an ancestor div of the column classes, and the CSS does the rest.

<div class="force-xs">
  <div class="row">
    <div class="col-lg-8">.col-lg-8</div>      
    <div class="col-lg-4">.col-lg-4</div>
    <div class="col-md-6">.col-md-6</div>      
    <div class="col-sm-2">.col-sm-2</div>
    <div class="col-xs-6">.col-xs-6</div>
    <div class="col-xs-4">.col-xs-4</div>
    <div class="col-xs-2">.col-xs-2</div>
  </div>
</div>

Works on combined .col-* classes the way you would expect

highlight shows active class

.col-sm-12 .col-xs-10
.col-sm-4 .col-xs-2
.col-md-8 .col-xs-6
.col-md-4 .col-xs-6
.col-lg-2 .col-xs-8
.col-lg-6 .col-xs-4
Even with many col classes: .col-lg-2 .col-md-6 .col-sm-8 .col-xs-10
Regardless of the order in which you declare them: .col-lg-2 .col-xs-10 .col-md-6 .col-sm-8

Works within modals which I'm pretty sure is the primary use case

The content in all three of these modals, including the col-classes, is identical. I've put the force-* class on the modal-body, but you can put it on any ancestor of the col-classes.

What doesn't work:

Offsets: It picks up the width from your declared device, but not the offset itself, so they end up all over the place

Pushes: Same probem as offsets

Any of the .hidden-* classes: Pretty straight-forward, so I should do this in an upcoming release
